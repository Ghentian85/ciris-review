# CIRIS Review — User Guide

A visual approval platform for photoshoots, lookbooks, e‑commerce and campaign
imagery. One place to upload, mark up, decide, and hand back to the post‑prod
pipeline.

Three roles, three short sections. Pick yours.

---

## Quick start — Admin / Studio

You set up the project, upload images, and pick who reviews.

### 1. Create a project
- Sign in at **review.ciris.studio**
- Click **+ New project** on the dashboard
- Name it (e.g. *Moods FW26*) and link a client — done

### 2. Upload images
- Open the project
- Click **Upload images** in the hero
- Drop in a batch — preview + thumbnail are generated automatically
- High‑res masters stay on your side; only review‑quality previews live here

### 3. Invite reviewers
- Open the **Settings drawer** (top‑right of the project hero)
- Go to the **Members** tab
- Enter email + role:
  - **Client reviewer** — sees images, comments, makes the call
  - **Post‑production** — gets the feedback digest, marks items addressed
  - **Internal reviewer** — full project access, can also invite others
  - **Admin** — internal reviewer + org‑level settings
- The invitee gets one email. Clicking signs them in **and** opens the project — no password setup unless they want one.

### 4. Open the round
- Once every image is decided internally and you're ready for the client, click **Notify client** on the project hero
- They receive a *Review is ready* email with a one‑click sign‑in link

### 5. After the client submits
- You get a digest email summarising the round — counts + per‑image comments
- Post‑production tackles the revision list
- When ready, upload v2 of each revised image (the upload screen matches the filename to the original slot)
- Click **Notify client** again to open the next round

### Admin tools (Settings → Members)
- **Reset** — emails the member a password‑reset link
- **Revoke** — removes them from the project (their account stays)

## Quick start — Client reviewer

You're here to look at images and tell the team what's working.

### Signing in
You'll get an invite email — click **Open the project**. That's it. The link signs you in.

You'll be asked once if you want to set a password (so signing in next time is one step). Either set one or skip — you can always click sign‑in links from later notification emails too.

### Reviewing an image
Click any image from the grid. The reviewer opens with:
- **The image** in the centre. Click any point to start a comment.
- **Annotation tools** at the top‑left:
  - 📍 **Pin** — drop a marker on a spot
  - 🟥 **Rectangle** — frame an area
  - ✏️ **Draw** — freehand circle / scribble
- **Comments + decision** on the right rail

#### Workflow
1. Pick the **Pin** tool (default), click on the part of the image you want to comment on
2. Type your note → **Save**
3. Repeat for each note
4. When done with this image, hit one of the two decision buttons:
   - **Needs revision** (red) — something must change
   - **Approve** (green) — looking good
5. Use the **← →** arrows or keyboard arrows to move to the next image

#### Useful while reviewing
- Hover a comment on the right → the marker on the image lights up (and vice versa)
- **F** for fullscreen — same controls, more space
- **+ / − / 0** to zoom in, out, reset
- Compare V1 vs V2 → click the **Compare** link in the version switcher

### Submitting your round
Once every image has a decision (no greys left in the *Still to review* tab):
- Click **Submit round** — the big CTA at the top
- The post‑prod team gets a digest with all your feedback in one email

### Coming back later
- Bookmark **review.ciris.studio** — sign in with email + password (or "Email me a sign‑in link")
- Notification emails always include a one‑click sign‑in URL — click and you're in

## Quick start — Post‑production

You get a digest after each round. Your job: address the feedback, upload v2.

### Find your work
- Sign in → **Work** tab in the top nav
- Lists every project with revision requests, sorted by who needs you most
- Click a project → opens directly on the *Needs revision* tab

### Working through an image
Open any image with revision comments.

- On the right rail: every client comment with its marker
- **Hover or click a comment** → the matching marker on the image lights up
- **Mark Done** — when you've handled a comment in your retouch, click the **square checkbox** on the right of the comment. It greys out + strikes through. The header shows *X done* so you can see your progress.
- The "Done" state syncs across the team — admins see it too

### Uploading v2
- Once retouched, head back to the project's **Upload** screen
- Drop the v2 file with the **same filename** as the original — the system matches it to the existing slot and creates a new version
- The image status flips back to *Pending* so the client re‑reviews

### When the client submits the next round
- You get a fresh digest
- Resolved comments from the previous round stay marked done (no need to re‑resolve)

## Reference

### Keyboard shortcuts in the reviewer

| Key | Action |
| --- | --- |
| **← →** | Previous / next image |
| **V** | Cursor tool |
| **P** | Pin tool |
| **R** | Rectangle tool |
| **D** | Draw tool |
| **F** | Toggle fullscreen |
| **+ / −** | Zoom in / out |
| **0** | Reset zoom |
| **Esc** | Close popover / exit fullscreen |

### Account settings
- Click your **email + avatar** top‑right
- Set or change your password
- Forgot your password? `/login` → **Forgot?** → email → reset link

### Share links (public previews)
Sometimes a stakeholder needs a peek without an account.
- Project hero → **Settings drawer** → **Share** tab → create link
- Optional password protection
- Optional expiry date
- The recipient sees a **read‑only gallery** — no comments, no decisions, no originals

### Versions
- Each image keeps every uploaded version
- Switch between them in the right rail's version pills
- **V2** badge = a fresh upload for revision; the client will see a *pending* status until they re‑review

### What to do if…
- **The sign‑in link from email looks broken (white space, line break)** — copy the whole URL (including the part after the break) and paste into your browser
- **You're invited but can't log in** — the invite link is single‑use. Ask the admin to **Reset** you from the Members panel — they'll send a fresh sign‑in link
- **You see "session expired"** — sign in again at `/login`. The session lasts 30 days

## Email cadence — what each role gets

| Email | Sent to | When |
| --- | --- | --- |
| **Invite** | New member | Admin invites them via Members panel |
| **Review ready** | Client | Admin clicks *Notify client* |
| **Round digest** | Post‑prod + admin | Client submits the round |
| **Round confirmed** | Client | Client submits the round (self‑confirmation) |
| **Password reset** | Anyone | They request it, or admin sends one |

Every email except the round‑confirmed includes a **one‑click sign‑in URL** valid for 14 days, so coming back is always one click.
